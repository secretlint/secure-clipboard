# クリップボードマネージャー由来のコピー検知 設計仕様

## 背景

現状の `ClipboardMonitor` は、クリップボード変化検知の瞬間に `NSWorkspace.shared.frontmostApplication` を見てコピー元アプリを特定し、そのbundle IDが `skipScanAppIdentifiers` に含まれていればスキャンをスキップする。

Alfred のクリップボード履歴から再コピーするケースでは、Alfred が dismiss された後にペーストボードへの書き込みが発生するため、`frontmost` には既に元アプリ（例: ターミナル）が戻っており、`skipScanAppIdentifiers` に `com.runningwithcrayons.Alfred` を入れていてもスキップが効かない。結果として、Alfred 履歴経由で再コピーしただけで毎回シークレットスキャンが走り、マスク済みの内容がさらに加工されるなど意図しない挙動になる。

1Password のようにコピー時点で frontmost になっているアプリは現状の判定で正しくスキップできているため、本改修の対象外。

## 要件

- Alfred のクリップボード履歴からの再コピーを `skipScanAppIdentifiers` で無視できる（Alfred 履歴経由のコピーには `com.runningwithcrayons.alfred.clipping` が付与されるため、このマーカーを skip 対象に入れれば検知できる）
- 既存の `skipScanAppIdentifiers` 設定だけで管理できる（新しい config キーを増やさない）
- 既存動作（frontmost による bundle ID 判定）は維持する
- 判定方針は「ログに現れる identifier と skip リストの完全一致」に限定する。識別情報が pasteboard に残らない操作まで無理に検知しない
- 将来、他のクリップボードマネージャ（Raycast / Paste / Maccy など）に、ユーザー設定だけで拡張できる

## 非対象

- `org.nspasteboard.TransientType` / `AutoGeneratedType` / `ConcealedType` のハンドリング
- 新しい config キーの追加
- SecureClipboard 側で「どのアプリがどの pasteboard type を書くか」の内部マッピングをメンテナンスすること（拡張はすべて config で済ませる方針）

## 調査結果

実機で `NSPasteboard` のメタデータをログに吐いて確認した内容（2026-04-20）。

### Alfred のクリップボード履歴から再コピー
```
frontmost=com.cmuxterm.app
nspasteboard.source=nil
types=[public.utf8-plain-text, NSStringPboardType, com.runningwithcrayons.alfred.clipping]
```

- frontmost は Alfred を呼び出す前のアプリに戻っている
- `org.nspasteboard.source` は書き込まれていない
- **`com.runningwithcrayons.alfred.clipping` という Alfred 独自の pasteboard type が必ず含まれる** → 再コピー判定の決定的なマーカー

### 通常のアプリからのコピー
Alfred 独自マーカーは付かない。`org.nspasteboard.source` も通常は無い。

## 設計

### `skipScanAppIdentifiers` の意味を拡張する

`skipScanAppIdentifiers` の各文字列を「スキップ対象の識別子」と解釈する。bundle ID に限定しない。SecureClipboard はクリップボード変化を検知した際に、次の3つの情報源を集めて、この配列と照合する。

1. `NSWorkspace.shared.frontmostApplication?.bundleIdentifier`
2. `pasteboard.string(forType: "org.nspasteboard.source")` の値（nspasteboard.org convention）
3. `pasteboard.types` の各 rawValue 文字列

1〜3 のいずれかが `skipScanAppIdentifiers` に含まれていればスキャンをスキップする。

この設計なら、SecureClipboard 側に内部マッピングを持たなくて済み、ユーザーはアプリごとに必要な識別子を設定に追加するだけで拡張できる。

### ユーザー設定例

```json
{
  "skipScanAppIdentifiers": [
    "com.1password.1password",
    "com.runningwithcrayons.Alfred",
    "com.runningwithcrayons.alfred.clipping"
  ]
}
```

| 識別子 | マッチ対象 | 用途 |
|---|---|---|
| `com.1password.1password` | frontmost bundle ID | 1Password のコピー（1Passwordがfrontmost） |
| `com.runningwithcrayons.Alfred` | frontmost bundle ID | Alfredがfrontmostの短時間に起きたコピー |
| `com.runningwithcrayons.alfred.clipping` | pasteboard type | Alfred 履歴からの再コピー（Alfredはdismiss済み） |

### sourceApp 表示

スキップされた場合は検知履歴に載らないため、本改修で `sourceApp` の導出ロジックは変更しない。既存の `NSWorkspace.shared.frontmostApplication?.localizedName` をそのまま使う。

### 実装配置

判定ロジックは `AppConfig` に追加するのが自然。既存 `shouldSkipScan(bundleId:)` は後方互換用に残し、新しい判定関数を追加する:

```swift
func shouldSkipScan(
    frontmostBundleId: String?,
    pasteboardTypes: [String],
    nspasteboardSource: String?
) -> Bool
```

`ClipboardMonitor` 側ではこの関数を呼び、スキップ判定を一元化する。

### デバッグログ

調査で仕込んだ `logger.info(...)` は `logger.debug(...)` に下げる。本番ビルドでは既定で出力されず、必要なときは `log stream --level debug` で確認できる。

## 影響範囲

- `SecureClipboard/AppConfig.swift`: 新 `shouldSkipScan(frontmostBundleId:pasteboardTypes:nspasteboardSource:)` を追加
- `SecureClipboard/ClipboardMonitor.swift`: 判定ロジックを新 API に差し替え、ログレベルを `.debug` に下げる
- `SecureClipboardTests/AppConfigTests.swift`: 新 API 用のテストを追加（既存 `shouldSkipScan(bundleId:)` のテストは維持）
- `README.md`: 設定例に Alfred 関連エントリを追加し、「pasteboard type もそのまま書ける」旨を簡潔に説明

既存 `AppConfig.skipScanAppIdentifiers` のスキーマと JSON フォーマットは変更しない。

## テスト

単体テスト（`AppConfigTests`）で以下をカバーする:

- `skipScanAppIdentifiers` に Alfred の独自 pasteboard type が含まれる。`pasteboardTypes` に同じ文字列が含まれる → skip
- `skipScanAppIdentifiers` に bundle ID が含まれる。`frontmostBundleId` が一致 → skip
- `skipScanAppIdentifiers` に bundle ID が含まれる。`nspasteboardSource` が一致 → skip
- どの情報源にも一致が無い → skipしない
- `skipScanAppIdentifiers` が nil → skipしない

## 確認手順（manual）

1. `bash scripts/build-app.sh` でビルド
2. `~/.config/secure-clipboard/config.json` の `skipScanAppIdentifiers` に以下を追加:
   - `com.runningwithcrayons.Alfred`
   - `com.runningwithcrayons.alfred.clipping`
3. Alfred のクリップボード履歴から、過去に検知されたシークレットを選んで再コピー
4. SecureClipboard の検知履歴にシークレット検知が記録されないことを確認
5. 通常のアプリからシークレットをコピーすると、従来通り検知・マスクされることを確認
